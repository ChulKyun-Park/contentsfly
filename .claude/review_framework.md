# 기획안 5관점 리뷰 프레임워크

기획안 작성이 완료되면 아래 5가지 관점에서 자체 리뷰를 수행하고, 각 항목별 평가 결과를 사용자에게 보고한다.

---

## 1. 정합성 (Consistency)
- 기획안 내부의 정보가 서로 모순되지 않는가?
- IA(정보 아키텍처), URL 체계, 메뉴 구조, 컴포넌트 목록이 일치하는가?
- 상수(constants.ts)와 실제 페이지 설계가 동기화되어 있는가?
- 디자인 토큰(색상, 타이포, 간격)이 일관되게 적용되어 있는가?

## 2. 완전성 (Completeness)
- 사용자가 요구한 모든 페이지/섹션/기능이 빠짐없이 포함되었는가?
- 각 페이지의 핵심 콘텐츠, CTA, 데이터 소스가 명시되어 있는가?
- 반응형 동작(모바일/태블릿/데스크톱)이 정의되어 있는가?
- 엣지 케이스(빈 데이터, 에러 상태, 로딩 상태)가 고려되었는가?
- SEO 메타데이터, OG 태그, 접근성 요구사항이 포함되었는가?

## 3. 실현가능성 (Feasibility)
- 선택한 기술 스택으로 기획된 기능을 구현할 수 있는가?
- 외부 의존성(API, 서드파티 서비스)이 현실적으로 연동 가능한가?
- 작업량이 스프린트 단위로 합리적으로 분배되어 있는가?
- 기존 코드베이스/디자인 시스템과의 호환성이 확인되었는가?

## 4. 사용자 경험 (User Experience)
- 핵심 사용자 플로우(방문 → 탐색 → 전환)가 자연스러운가?
- CTA 배치와 문구가 전환 목표에 부합하는가?
- 정보 계층이 명확하고 스캔하기 쉬운가?
- 로딩/전환 애니메이션이 과하지 않고 목적에 부합하는가?

## 5. 비즈니스 정렬 (Business Alignment)
- 기업 고객(B2B) + 개인 크리에이터(B2C) 듀얼 타겟이 균형있게 반영되었는가?
- 경쟁사 대비 차별화 포인트가 명확히 드러나는가?
- 가격 정보, 프로세스, 신뢰 요소(소셜프루프)가 전환에 기여하는가?
- 장기 로드맵(플랫폼 확장, 백엔드 연동)과의 연결 고리가 있는가?

---

## 리뷰 출력 형식

```
### 기획안 5관점 리뷰 결과

| 관점 | 상태 | 요약 |
|------|------|------|
| 정합성 | ✅/⚠️/❌ | 한 줄 요약 |
| 완전성 | ✅/⚠️/❌ | 한 줄 요약 |
| 실현가능성 | ✅/⚠️/❌ | 한 줄 요약 |
| 사용자 경험 | ✅/⚠️/❌ | 한 줄 요약 |
| 비즈니스 정렬 | ✅/⚠️/❌ | 한 줄 요약 |

**개선 필요 사항:**
- (있을 경우 구체적으로 나열)

→ 사용자 승인 요청
```

- ✅ = 문제 없음
- ⚠️ = 경미한 이슈 (진행 가능하나 개선 권장)
- ❌ = 블로커 (해결 후 진행 필요)
